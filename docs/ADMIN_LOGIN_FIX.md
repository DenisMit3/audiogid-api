# Исправление проблемы входа в админку

## Проблема
При попытке входа в админку (`http://82.202.159.64:3080/login`) возникает ошибка 400 (Bad Request).

## Диагностика

### Проверка API
API endpoint работает корректно:
```bash
curl -X POST "http://82.202.159.64/v1/auth/login/email" \
  -H "Content-Type: application/json" \
  -d '{"email":"mit333@list.ru","password":"Solnyshko3"}'
```

**Результат:** ✅ Возвращает токены успешно

### Проблема в админке
Ошибка возникает в `apps/admin/app/api/auth/login/route.ts` при обработке запроса от фронтенда.

## Решение

### 1. Улучшена обработка ошибок в login route
Добавлена проверка валидности JSON и улучшена обработка ошибок.

### 2. Проверка CORS
Убедитесь, что в `apps/api/index.py` добавлен origin для админки:
```python
"http://82.202.159.64:3080",  # Cloud.ru Admin
```

### 3. Проверка переменных окружения
Убедитесь, что админка использует правильный API URL:
- `NEXT_PUBLIC_API_URL=http://82.202.159.64/v1` (если не установлена, используется по умолчанию)

## Шаги для проверки

1. **Проверить логи админки на сервере:**
   ```bash
   ssh user@82.202.159.64
   sudo journalctl -u audiogid-admin -f
   ```

2. **Проверить логи API:**
   ```bash
   sudo journalctl -u audiogid-api -f
   ```

3. **Проверить доступность API из админки:**
   - Откройте консоль браузера (F12)
   - Проверьте Network tab при попытке входа
   - Посмотрите точный URL запроса и ответ сервера

## Возможные причины ошибки 400

1. **Неправильный формат JSON** - исправлено добавлением try-catch
2. **CORS проблемы** - проверьте настройки CORS в API
3. **Проблемы с proxy** - проверьте, что Next.js API route работает
4. **Проблемы с переменными окружения** - проверьте `NEXT_PUBLIC_API_URL`
